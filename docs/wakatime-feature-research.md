## Features Detail

### Core Tracking
- **Automatic time tracking** - Detects typing start/stop in editors. No manual timers needed.
- **Heartbeat system** - Plugins send heartbeats on keystrokes; joined into durations via configurable timeout.
- **AFK detection** - Stops tracking when user stops typing.
- **Accuracy** - Accurate to 1 microsecond (0.000001 seconds).
- **Offline support** - Plugins queue heartbeats when offline and sync when back online.

### Metrics Breakdown
Summaries are broken down by:
- **Projects** - Auto-detected from git repos, IDE workspace, or folder structure
- **Files** - Individual file tracking
- **Branches** - Time per branch (feature/bugfix tracking)
- **Languages** - 600+ languages supported
- **Editors** - 90+ editors/IDEs supported
- **Operating Systems** - Windows, macOS, Linux
- **Categories** - coding, building, indexing, debugging, browsing, running tests, writing tests, manual testing, writing docs, communicating, code reviewing, notes, researching, learning, designing, ai coding
- **Machines** - Track across multiple devices
- **Dependencies** - Library/package usage tracking

### AI Code Tracking
- Track lines of code generated by AI in repos
- `ai_line_changes` and `human_line_changes` fields on heartbeats
- `ai coding` category for AI-assisted coding sessions

### Goals System
- Set coding targets (e.g., "code X hours per day/week")
- Email reminders
- Share goals with friends for accountability
- Chart data showing progress over time
- Free: 1 goal, Basic: 3, Premium+: unlimited

### Leaderboards
- **Public leaderboard** at /leaders - ranked by coding activity (7-day window)
- Filterable by language, country, hireable status
- **Private leaderboards** for friends/teams
- Language-specific leaderboards

### Commit & PR Metrics
- Overlay WakaTime stats onto git commit history
- Time spent per commit
- Time spent per pull request
- Supports GitHub, GitLab, Bitbucket

### Team Dashboards
- Aggregate stats across developers
- Per-project, per-file, per-branch breakdowns
- Privacy controls (hide specific projects, anonymous mode)
- CSV/JSON export
- Alerts & reporting via email or Slack

### Sharing & Embeds
- **Embeddable SVG charts** for websites
- **Embeddable JSON** with JSONP support (no CORS on main API)
- **Project/repo badges** for GitHub READMEs
- **Revocable** unique URLs for each embed
- Never exposes API key

### Custom Rules
- Remap file paths to different projects
- Change detected languages
- Operations: equals, contains, starts with, ends with
- Can be applied retroactively to past stats

### Insights
- Weekday analysis
- Best coding day
- Daily averages
- Per-project/language/editor/category/machine/OS breakdowns
- Available for various time ranges

### External Durations
- Import time from external sources (Google Calendar, Microsoft Calendar, Zoom)
- Separate from heartbeat-tracked time
- Bulk creation (up to 1000 per request)

### Data Export
- Full data dumps (daily summaries or raw heartbeats)
- Email notification when export is ready

### Organizations
- Organization management with dashboards
- Per-member summaries and durations
- Organization-level custom rules
- Privacy controls (default_project_privacy, is_duration_visible)
- Permission system (can_list_dashboards, can_create_dashboards, etc.)

### Integrations (1st-party)
- **GitHub** - time per commit/issue/PR
- **GitLab** - time per commit/issue/PR
- **Bitbucket** - time per commit/issue/PR
- **Google Calendar** - import meetings
- **Microsoft Calendar** - import Outlook meetings
- **Office 365** - share commits with team
- **Slack** - share commits with coding time
- **Zoom** - meeting time on dashboard
- **Paymo** - sync with timesheets
- **Zube** - time per ticket

### Supported Editors (90+)
VS Code, IntelliJ IDEA, PyCharm, Vim, Neovim, Emacs, Sublime Text, Visual Studio, Xcode, Eclipse, Android Studio, Cursor, Windsurf, Claude Code, Figma, Blender, Chrome, Firefox, Terminal, WebStorm, GoLand, Rider, PhpStorm, RubyMine, DataGrip, CLion, Notepad++, Nova, Zed, and many more.

Desktop apps available for Windows and Mac for tracking unsupported applications.

## API Overview

### Authentication
1. **API Key** - HTTP Basic Auth with base64(`apikey:`) or `?api_key=` query param
2. **Bearer Token** - `Authorization: Bearer {token}`
3. **OAuth 2.0** - Full flow with authorize/token/revoke endpoints
   - Token via `response_type=code`: expires 365 days
   - Token via `response_type=token`: expires 12 hours
   - Max 8 active tokens per user per app

### OAuth Scopes
`read_summaries` (with sub-scopes: .categories, .dependencies, .editors, .languages, .machines, .operating_systems, .projects), `read_stats` (with sub-scopes: .best_day, .categories, .dependencies, .editors, .languages, .machines, .operating_systems, .projects), `read_goals`, `read_orgs`, `write_orgs`, `read_private_leaderboards`, `write_private_leaderboards`, `read_heartbeats`, `write_heartbeats`, `email`

### Rate Limiting
- 10 requests/second averaged over any 5-minute period
- Exceeding returns 429 Too Many Requests (or sometimes 302 Redirect)

### Heartbeat Fields

**Required:**
- `entity` (string) - File path or app name
- `type` (string) - `file`, `app`, or `domain`
- `time` (number) - UNIX epoch float

**Optional:**
- `category` - coding, building, indexing, debugging, browsing, running tests, writing tests, manual testing, writing docs, communicating, code reviewing, notes, researching, learning, designing, ai coding
- `project` (string)
- `project_root_count` (number)
- `branch` (string)
- `language` (string)
- `dependencies` (string)
- `lines` (number)
- `ai_line_changes` (number)
- `human_line_changes` (number)
- `lineno` (number)
- `cursorpos` (number)
- `is_write` (boolean)

### Summary Response Categories
Each daily summary contains breakdowns for: categories, projects, languages, editors, operating_systems, dependencies, machines, branches, entities

### HTTP Response Codes
- `200` OK
- `201` Created
- `202` Accepted (stats still calculating; heartbeat queued)
- `400` Bad Request
- `401` Unauthorized
- `403` Forbidden
- `404` Not Found
- `429` Too Many Requests
- `500` Server Error
